# QTR AI v2 Architecture

## Общая архитектура

QTR AI v2 строится по трехуровневой архитектуре.

```
            Анализ рынка
                  │
                  ▼
         Логика стратегии
                  │
                  ▼
      Исполнение торговых решений
```

---

# Уровень 1. Анализ рынка (Market Analysis)

Назначение:

Получить объективное состояние рынка.

На этом уровне не существует понятий BUY или SELL.

Модули:

- Collector
- Indicators
- Market Structure
- Market Context

Результат:

Полностью описанное состояние рынка.

---

# Уровень 2. Логика стратегии (Strategy Layer)

Назначение:

Определить наличие торгового сетапа и принять решение.

Модули:

- Setup Engine
- Decision Engine
- Strategy Engine

Результат:

Торговый сигнал.

```
BUY

SELL

WAIT
```

---

# Уровень 3. Исполнение (Execution Layer)

Назначение:

Открытие и сопровождение сделки.

Модули:

- Entry Engine
- Risk Manager
- Broker
- Journal
- Telegram

Результат:

Полностью сопровождаемая сделка.

---

# Главный принцип

Каждый уровень использует только данные предыдущего уровня.

Ни один модуль не имеет права вычислять то, что уже рассчитано другим модулем.

Пример:

Market Structure определяет BOS.

Decision Engine использует BOS.

Decision Engine не имеет права вычислять BOS самостоятельно.

# Правила разработки

Каждый новый модуль обязан иметь:

- четкое назначение;
- входные данные;
- выходные данные;
- отдельный тест.

Модуль считается завершенным только после успешного прохождения тестирования.
